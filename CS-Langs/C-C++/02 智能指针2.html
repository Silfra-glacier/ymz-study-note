智能指针2 ​

与智能指针一样，这一次介绍std::shared_ptr。

std::shared_ptr ​

与unique_ptr不同在于多个共享指针可以指向同一片内存区域。

std::make_shared ​

与make_unique相同，用于申请内存空间，但他不可以申请数组空间，要申请数组空间，请使用new。

相同类型的共享指针可以相互赋值了：

c++
ptr1 = ptr2;

ptr.use_count() ​

由于多个共享指针可以指向同一片内存，因此共享指针提供判定当前有多少指针指向该内存区域的方法，使用use_count()查看，返回值为 long类型。

c++
std::shared_ptr<int>ptr3{ std::make_shared<int>(100) };
std::shared_ptr<int>ptr4{ };
ptr4 = ptr3;
std::cout << ptr3.use_count() << std::endl;
std::cout << ptr4.use_count() << std::endl;  // 二者是相同的
// 输出为2

ptr.unique() ​

用于查看当前共享指针是否为指向该内存区域的最后一个指针。返回 bool类型。

c++
std::cout << std::boolalpha << ptr.unique() << std::endl;

该方法在C14后被废弃，使用旧版C标准以执行。

ptr.reset() ​

将当前共享指针置空，如果该指针是最后一个指针，还释放内存空间，否则不释放。

代码： ​

c++
#include <iostream>

int main()
{
	int* a{};
	std::shared_ptr<int>ptr{};
	std::cout << ptr << std::endl;

	std::shared_ptr<int[]>ptr2{ new int[100] };
	std::cout << ptr2 << std::endl;

	std::shared_ptr<int>ptr3{ std::make_shared<int>(100) };
	std::cout << ptr3 << std::endl;

	std::shared_ptr<int>ptr4{  };
	//std::cout << ptr4 << std::endl;

	ptr4 = ptr3;
	std::cout << ptr4 << std::endl;

	std::cout << ptr3.use_count() << std::endl;  // 返回值 long 
	std::cout << std::boolalpha << ptr3.unique() << std::endl;  // 返回值 bool 
	std::cout << std::boolalpha << ptr2.unique() << std::endl; 

	ptr4.reset();  // 将 ptr4 置空，若该指针是唯一指针，则还释放空间，否则不释放
	std::cout << std::boolalpha << ptr3.unique() << std::endl;
	std::cout << ptr4 << std::endl;
	std::cout << ptr3 << std::endl;


}
